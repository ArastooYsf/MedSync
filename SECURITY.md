| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it as soon as possible.  
We will review and address the issue as quickly as possible. Follow the steps below:

1. **Submit an Issue**: Open an issue in the repository with the "Security" tag.
2. **Include Details**: Provide as many details as possible to help us understand the vulnerability, including code examples, screenshots, and reproduction steps.
3. **Expected Response Time**: You can expect a response within 48 hours, and we will keep you updated on our progress.
4. **Accepted/Declined**: If the vulnerability is accepted, we will work on a fix and release an update. If declined, we will explain why and offer advice on how to mitigate the issue.

## گزارش آسیب‌پذیری

اگر یک آسیب‌پذیری امنیتی در این پروژه کشف کردید، لطفاً فوراً آن را گزارش دهید.  
ما به سرعت موضوع را بررسی و حل خواهیم کرد. مراحل زیر را دنبال کنید:

1. **ایجاد یک Issue**: یک Issue جدید در مخزن باز کرده و برچسب "Security" را اضافه کنید.
2. **شامل کردن جزئیات**: تا حد امکان جزئیات کاملی از آسیب‌پذیری را ارائه دهید، از جمله نمونه‌های کد، اسکرین‌شات‌ها و مراحل تکثیر.
3. **زمان پاسخ‌دهی پیش‌بینی‌شده**: شما می‌توانید ظرف ۴۸ ساعت پاسخ دریافت کنید و در طول روند رفع مشکل، ما شما را به‌روز خواهیم کرد.
4. **پذیرفته یا رد‌شده**: اگر آسیب‌پذیری پذیرفته شود، ما به‌سرعت روی رفع آن کار خواهیم کرد و یک به‌روزرسانی منتشر خواهیم کرد. اگر رد شود، دلیل آن را توضیح خواهیم داد و راه‌حل‌های ممکن برای کاهش خطر را پیشنهاد خواهیم داد.

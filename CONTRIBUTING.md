# Contributing to MedSync

Welcome! We're excited that you're interested in contributing to **MedSync**.

We believe in building a strong community around this project, and every contribution, no matter how big or small, is appreciated. 💙

---

## How to Contribute

### 1. Fork the Repository

- Click the **Fork** button on the top right of the repository page.
- Clone your forked repository:
  ```bash
  git clone https://github.com/your-username/MedSync.git
  ```

### 2. Create a Branch

- Create a new branch for your feature or fix:
  ```bash
  git checkout -b feature/your-feature-name
  ```

### 3. Make Your Changes

- Write clear, clean, and well-documented code.
- Follow the existing style and patterns.

### 4. Test Your Changes

- Make sure your changes don't break existing functionality.
- Test thoroughly!

### 5. Commit and Push

- Write meaningful commit messages:
  ```bash
  git add .
  git commit -m "Add feature: description"
  git push origin feature/your-feature-name
  ```

### 6. Create a Pull Request

- Open a pull request from your branch to the `main` branch.
- Provide a clear description of the changes.
- Reference related issues if applicable.

---

## Contribution Guidelines

- Follow the **Code of Conduct**.
- Keep pull requests small and focused.
- Write meaningful descriptions and comments.
- Be respectful and collaborative.

---

## Need Help?

If you have any questions or ideas, feel free to open an issue!

Let's make **MedSync** better together. 🚀

# نحوه مشارکت در MedSync

به پروژه‌ی **MedSync** خوش آمدید! ما خوشحالیم که مایل به همکاری هستید. 💙

ما به ایجاد یک جامعه‌ی قوی حول این پروژه باور داریم و از هرگونه مشارکت، چه کوچک چه بزرگ، استقبال می‌کنیم.

---

## چگونه مشارکت کنید

### ۱. پروژه را Fork کنید

- روی دکمه **Fork** در بالای صفحه مخزن کلیک کنید.
- مخزن Fork شده را کلون کنید:
  ```bash
  git clone https://github.com/your-username/MedSync.git
  ```

### ۲. یک شاخه (Branch) بسازید

- برای ویژگی یا اصلاح خود یک شاخه جدید ایجاد کنید:
  ```bash
  git checkout -b feature/your-feature-name
  ```

### ۳. تغییرات خود را اعمال کنید

- کدی تمیز، واضح و مستندسازی شده بنویسید.
- از الگوها و سبک کد پروژه پیروی کنید.

### ۴. تغییرات را تست کنید

- مطمئن شوید که تغییرات شما باعث شکسته شدن عملکردهای موجود نمی‌شود.
- به خوبی تست کنید!

### ۵. Commit و Push کنید

- پیام‌های Commit معنادار بنویسید:
  ```bash
  git add .
  git commit -m "افزودن ویژگی: توضیح"
  git push origin feature/your-feature-name
  ```

### ۶. Pull Request ایجاد کنید

- از شاخه خود به شاخه `main` درخواست Pull باز کنید.
- توضیح روشنی درباره تغییرات بدهید.
- در صورت وجود، به مسائل (Issues) مرتبط اشاره کنید.

---

## قوانین مشارکت

- از **قانون رفتار (Code of Conduct)** پیروی کنید.
- Pull Request‌ها را کوچک و متمرکز نگه دارید.
- توضیحات و کامنت‌های معنادار بنویسید.
- محترمانه و همکاری‌کننده باشید.

---

## نیاز به کمک دارید؟

اگر سوال یا ایده‌ای داشتید، می‌توانید یک Issue باز کنید!

بیایید با هم **MedSync** را بهتر کنیم! 🚀
